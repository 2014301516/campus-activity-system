# 校园活动管理系统 项目目录与代码说明

> 用途：这份文档用于快速熟悉项目目录结构和主要代码文件，方便答辩前查阅。
>
> 阅读建议：先看“项目总览”，再看“前端目录”“后端目录”“数据库与文档目录”，最后看“答辩时重点讲哪些文件”。

---

## 1. 项目总览

项目根目录：`e:\keshe\campus-activity-system`

这个项目可以拆成 4 块：

1. `frontend`：前端页面和交互
2. `backend`：后端接口和业务逻辑
3. `docs`：数据库脚本和正式文档
4. `report`：答辩准备和课程报告整理

一句话理解：

- 前端负责“展示和点击”
- 后端负责“业务和权限”
- 数据库负责“存数据”
- 文档负责“说明项目和答辩准备”

## 3. 前端目录 `frontend`

前端技术栈：

- Vue 3
- Vite
- Element Plus
- Vue Router
- Pinia
- Axios
- ECharts

前端目录主要负责：

- 展示页面
- 响应用户点击
- 调用后端接口
- 显示数据和提示信息

### 3.4 前端接口层 `src/api`

#### [request.js](file:///e:/keshe/campus-activity-system/frontend/src/api/request.js)

作用：

- 对 Axios 进行统一封装
- 自动给请求带上 token
- 统一处理接口报错
- 401 时自动跳回登录页

这个文件很重要，因为它体现了前端如何和后端通信。

#### [index.js](file:///e:/keshe/campus-activity-system/frontend/src/api/index.js)

作用：

- 把所有接口按模块封装好
- 页面不直接写 URL，而是通过这里调用

常见模块包括：

- 用户接口
- 活动接口
- 报名接口
- 签到接口
- 评价接口
- 分类接口
- 公告接口
- 统计接口
- 管理员接口

### 3.5 前端路由 `src/router`

#### [index.js](file:///e:/keshe/campus-activity-system/frontend/src/router/index.js)

作用：

- 定义页面路由
- 定义角色访问权限
- 通过前置守卫控制页面跳转

这个文件能体现三件事：

1. 项目有哪些页面
2. 每个页面给谁用

页面对应关系大致如下：

- `/login`：登录页
- `/home`：首页
- `/activity/:id`：活动详情
- `/my-activities`：学生的我的报名
- `/manage`：组织者活动管理
- `/admin`：管理员后台
- `/profile`：个人中心

## 7. 整个项目怎么串起来理解

可以按下面这条链理解：

```text
用户在前端页面点击
-> 前端 views 调用 api
-> api 通过 request.js 请求后端
-> 后端 controller 接收请求
-> service 处理业务逻辑
-> mapper 操作数据库
-> entity 对应数据库表
-> Result 统一返回给前端
```

所以：

- 前端是“用户看到的”
- 后端是“系统在做的”
- 数据库是“系统记住的”
