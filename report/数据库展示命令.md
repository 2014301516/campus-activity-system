# 答辩现场 — MySQL 命令行展示数据库

**使用方法**：打开终端，按顺序敲下面命令。每敲一条，给老师看一下输出，然后简要说明。

**前置命令（先执行一次）**：
```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus-activity-system
```
或者不在 mysql 交互模式下，直接在终端一条条执行也行：
```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SQL语句"
```

---

## 1. 查看所有表

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SHOW TABLES;"
```

> "系统一共有 7 张核心数据表：users 用户表、categories 分类表、activities 活动表、registrations 报名表、sign_ins 签到表、reviews 评价表、notices 公告表。"

---

## 2. 查看用户表

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT id, username, real_name AS 姓名, role AS 角色, CASE status WHEN 1 THEN '正常' ELSE '禁用' END AS 状态 FROM users ORDER BY id;"
```

> "6 个测试用户，三种角色：admin 是管理员，organizer01 和 02 是组织者，student01-03 是学生。密码全部用 BCrypt 加密存储，不可逆。"

---

## 3. 查看活动分类

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT id, name AS 分类名称, description AS 描述 FROM categories ORDER BY id;"
```

> "5 种分类：学术讲座、文体活动、志愿服务、学科竞赛、社团活动。管理员可以在后台增删分类。"

---

## 4. 查看活动列表

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT id, title AS 活动名称, status AS 状态, current_participants AS 已报名, max_participants AS 上限, DATE_FORMAT(start_time,'%m/%d %H:%i') AS 开始时间 FROM activities ORDER BY status, id;"
```

> "26 条活动数据，涵盖 6 种状态。pending 是待审核，approved 是已通过（学生可见），ongoing 是进行中，ended 是已结束，rejected 是已驳回，cancelled 是已取消。"

---

## 5. 查看每种状态各有多少

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT status AS 状态, COUNT(*) AS 数量 FROM activities GROUP BY status ORDER BY COUNT(*) DESC;"
```

> "可以看到状态分布：进行中和已通过的占多数，待审核和驳回的是少数——对应后台仪表盘的饼图数据来源。"

---

## 6. 查看报名记录

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT r.id, u.real_name AS 学生, a.title AS 活动, r.status AS 状态, DATE_FORMAT(r.registered_at,'%m/%d %H:%i') AS 报名时间 FROM registrations r JOIN users u ON r.user_id=u.id JOIN activities a ON r.activity_id=a.id ORDER BY r.id LIMIT 10;"
```

> "报名记录关联了用户表和活动表。这里可以看到谁报名了什么活动，状态是 registered 还是 cancelled。"

---

## 7. 查看签到记录

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT s.id, u.real_name AS 学生, a.title AS 活动, DATE_FORMAT(s.sign_in_time,'%m/%d %H:%i') AS 签到, IFNULL(DATE_FORMAT(s.sign_out_time,'%m/%d %H:%i'),'未签退') AS 签退 FROM sign_ins s JOIN users u ON s.user_id=u.id JOIN activities a ON s.activity_id=a.id ORDER BY s.id;"
```

> "签到记录关联了报名记录、用户、活动三张表。既有签到时间也有签退时间。"

---

## 8. 查看评价

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT r.id, u.real_name AS 学生, a.title AS 活动, r.rating AS 评分, r.comment AS 评价内容 FROM reviews r JOIN users u ON r.user_id=u.id JOIN activities a ON r.activity_id=a.id ORDER BY r.id;"
```

> "评分 1-5 星加文字评价。学生只能评价自己报过名的活动，且不能重复评价。"

---

## 9. 查看公告

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SELECT id, title AS 标题, content AS 内容, DATE_FORMAT(created_at,'%m/%d %H:%i') AS 发布时间 FROM notices ORDER BY id;"
```

> "管理员发布的公告，所有用户的首页顶部都能看到。"

---

## 10. 查看表结构（选一个展示设计细节）

```bash
mysql -u root -p123 --default-character-set=utf8mb4 campus_activity -e "SHOW CREATE TABLE activities\G" | head -20
```

> "可以看到每个字段的类型、约束、注释。status 字段用 VARCHAR(20) 存状态枚举，current_participants 和 max_participants 配合原子更新防超卖。"
