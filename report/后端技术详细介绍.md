# 校园活动管理系统 — 后端技术详细介绍

## 技术概览

| 技术 | 版本 | 作用 |
|------|------|------|
| Spring Boot | 2.7.18 | Java Web 框架，整合所有组件 |
| MyBatis-Plus | 3.5.5 | ORM 框架，操作数据库 |

## 一、Spring Boot

### 是什么

Spring Boot 是 Java 的 Web 开发框架。它在 Spring 基础上做了"自动配置"——以前用 Spring 需要写一堆 XML 配置文件，Spring Boot 把这些都简化了，开箱即用。

### 为什么用它

传统 Java Web 开发（Servlet / SSM）需要：
- 手动配置 web.xml
- 手动配置 Spring 的 applicationContext.xml
- 手动配置 MyBatis 的 SqlSessionFactory
- 手动配置事务管理器
- 手动部署 Tomcat

**Spring Boot 把上面五步压缩成一步**：引入 `spring-boot-starter-web`，直接写业务代码，内嵌 Tomcat 自动启动。

### 项目里的用法

`pom.xml` 通过 starter 引入各种能力：

| Starter | 提供的功能 |
|---------|-----------|
| `spring-boot-starter-web` | Spring MVC + 内嵌 Tomcat，几行注解就能写 RESTful API |
| `spring-boot-starter-security` | 认证和授权框架 |
| `spring-boot-starter-validation` | `@Valid` 参数校验 |

`application.yml` 集中管理所有配置——数据库连接、端口、JWT 密钥，不用 XML。

`CampusActivityApplication.java` 一个 `main` 方法启动整个项目：
```java
@SpringBootApplication
@MapperScan("com.cas.mapper")
public class CampusActivityApplication {
    public static void main(String[] args) {
        SpringApplication.run(CampusActivityApplication.class, args);
    }
}
```

### 答辩要点
- "约定优于配置"——省掉大量 XML，专注业务代码
- 内嵌 Tomcat——打成 jar 包直接运行，不需要单独部署服务器
- Starter 机制——引入即用，自动装配

### Spring Boot 和 SSM 的关系

老师要求里写了"SSM/Spring Boot+MyBatis"。SSM = Spring + Spring MVC + MyBatis，是 Spring Boot 出现之前的主流组合。Spring Boot 本质上是 SSM 的"自动化版本"——底层还是 Spring MVC + MyBatis，只是省掉了手动配置。**我们用的是 Spring Boot，底层就是 SSM**。

## 二、MyBatis-Plus

### 是什么

MyBatis-Plus 是 MyBatis 的增强工具。MyBatis 是 Java 操作数据库的框架，需要手写 SQL（可以写在 XML 或注解里）。MyBatis-Plus 在此基础上**提供了通用 CRUD、条件构造器、分页插件等开箱即用的功能。**

### 为什么用它（而不用 JPA 或手写 SQL）

| 对比 | MyBatis（原始） | JPA（Spring Data） | MyBatis-Plus（我们） |
|------|:--:|:--:|:--:|
| 简单 CRUD | 要写 SQL 或 XML | 自动（方法命名约定） | **自动（BaseMapper）** |
| 复杂查询 | 手写 SQL（灵活） | JPQL（不灵活） | **LambdaQueryWrapper（灵活+安全）** |
| 分页 | 手写 count+limit | Pageable | **插件自动生成** |
| 学习成本 | 中 | 高 | **低** |

### 项目里的用法

**1. BaseMapper 通用 CRUD**：

```java
// 只需继承，自动获得增删改查
public interface UserMapper extends BaseMapper<User> {
    // 不用写任何方法，BaseMapper 全部提供
}

// 使用
userMapper.selectById(1L);              // 查单个
userMapper.selectList(wrapper);         // 条件查询
userMapper.selectPage(page, wrapper);   // 分页查询
userMapper.insert(user);                // 新增
userMapper.updateById(user);            // 更新
userMapper.deleteById(1L);              // 删除
```

**2. LambdaQueryWrapper 类型安全的条件查询**：
```java
// 不需要写 SQL 字符串，Lambda 表达式编译期就检查字段名
LambdaQueryWrapper<Activity> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Activity::getCategoryId, query.getCategoryId())    // = 某个分类
       .like(Activity::getTitle, query.getKeyword())          // LIKE 关键词
       .in(Activity::getStatus, "approved", "ongoing")        // IN (...)
       .orderByDesc(Activity::getCreatedAt);                  // ORDER BY 倒序
this.page(pageParam, wrapper);
```

对比手写 SQL 字符串 `wrapper.eq("category_id", ...)`，Lambda 写法 `Activity::getCategoryId` 的好处是：**字段名改了你编译就报错，手写字符串要上线才炸。**

**3. 分页插件**：
```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
    return interceptor;
}
```
注册后，调用 `this.page(pageParam, wrapper)` 自动生成正确的 count + limit SQL，不用手写分页。

**4. 原子更新（防止并发超卖）**：

```java
// 用 setSql 做原子加减，防止报名人数超限
activityService.lambdaUpdate()
    .setSql("current_participants = current_participants + 1")
    .eq(Activity::getId, activityId)
    .lt(Activity::getCurrentParticipants, activity.getMaxParticipants())  // 防超卖
    .update();
```
这一行 SQL 翻译成：`UPDATE activities SET current_participants = current_participants + 1 WHERE id = ? AND current_participants < max_participants`。数据库层面保证原子性，不会超卖。

### 答辩要点
- BaseMapper：免写 CRUD SQL，省 90% 重复代码
- LambdaQueryWrapper：类型安全的动态查询，编译期防错
- 分页插件：一行注册，全项目生效
- 原子更新：`setSql` 防并发超卖，不用加锁

## 三、Spring Security

### 是什么

Spring Security 是 Spring 生态的安全框架。它做两件事：**认证**（你是谁）和**授权**（你能做什么）。

### 项目里的用法

**SecurityConfig.java 集中管理所有安全规则**：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf().disable()                    // 前后端分离，关闭 CSRF
            .cors().and()                        // 允许跨域
            .sessionManagement()
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)  // 无状态
            .and()
            .authorizeRequests()
                // 公开接口
                .antMatchers("/api/user/register", "/api/user/login").permitAll()
                .antMatchers(HttpMethod.GET, "/api/activity/**").permitAll()
                // 管理员接口
                .antMatchers("/api/admin/**").hasRole("ADMIN")
                // 组织者 + 管理员
                .antMatchers(HttpMethod.POST, "/api/activity")
                    .hasAnyRole("ORGANIZER", "ADMIN")
                // 其余接口需登录
                .anyRequest().authenticated()
            .and()
            .addFilterBefore(jwtAuthFilter, ...);  // JWT 过滤器

        return http.build();
    }
}
```

**权限分层**：

| 接口 | 谁可以访问 |
|------|-----------|
| `/api/user/register`, `/api/user/login` | 所有人 |
| `GET /api/activity/**` | 所有人（浏览活动不用登录） |
| `GET /api/category/**` | 所有人 |
| `POST /api/activity/{id}/register` | 已登录用户 |
| `POST /api/activity` | 组织者、管理员 |
| `/api/admin/**` | 仅管理员 |

**密码加密**：
```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();  // BCrypt 加密，不可逆
}
```

### 答辩要点
- 认证和授权分离——SecurityConfig 集中管理规则
- 无状态会话——不依赖 Session，适配前后端分离
- BCrypt 加密——数据库不存明文密码
- 路径级权限控制——按角色精确到具体 API

## 四、JWT (JSON Web Token)

### 是什么

JWT 是一串加密的字符串，用户登录后服务器发给他，之后每次请求带着它就能证明"我是谁"，不需要反复输密码。

### 它长什么样

```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiI0IiwidXNlcm5hbWUiOiJzdHVkZW50MDEiLCJyb2xlIjoic3R1ZGVudCJ9.xN_hE8G...
```

三个部分，用 `.` 分隔：

| 部分 | 内容 | 说明 |
|------|------|------|
| Header | `{"alg":"HS256"}` | 加密算法 |
| Payload | `{"userId":4,"role":"student"}` | 存了用户 ID 和角色，不存密码 |
| Signature | `xN_hE8G...` | 用密钥算出来的校验码，防止篡改 |

### 认证流程

```
1. 学生输入用户名密码 → POST /api/user/login
2. 后端校验通过 → JwtUtil.generateToken(userId, username, role) 生成 Token
3. 前端收到 Token → 存 localStorage → 跳首页
4. 之后每次发请求 → Axios 拦截器自动在 Header 加: Authorization: Bearer <token>
5. JwtAuthFilter 拦截请求 → 解析 Token → 验证 → 把 userId+role 放入 SecurityContext
6. Controller 通过 SecurityUtil.getCurrentUserId() 拿到当前用户
```

### 项目里的实现

**JwtUtil.java**：

```java
// 生成 Token
public String generateToken(Long userId, String username, String role) {
    return Jwts.builder()
        .setSubject(String.valueOf(userId))         // 存用户 ID
        .claim("username", username)
        .claim("role", role)
        .setExpiration(new Date(now + 86400000))    // 24 小时过期
        .signWith(key, SignatureAlgorithm.HS256)     // HMAC-SHA256 签名
        .compact();
}
```

**JwtAuthFilter.java**：
```java
// 每次请求来 → 从 Header 取 Token → 解析 → 设置 SecurityContext
String token = request.getHeader("Authorization").replace("Bearer ", "");
Claims claims = jwtUtil.getClaimsFromToken(token);
Long userId = Long.valueOf(claims.getSubject());
String role = claims.get("role", String.class);
// 注入 Spring Security 上下文
SecurityContextHolder.getContext().setAuthentication(authentication);
```

### 为什么用 JWT 而不是 Session

| | Session | JWT |
|------|------|------|
| 状态存储 | 服务端存 | **客户端存（无状态）** |
| 扩展性 | 多服务器需共享 Session | **天然支持分布式** |
| 前后端分离 | Cookie 有跨域限制 | **Header 传，无跨域问题** |
| 服务器压力 | 每次查 Session 表 | **Token 自包含信息，不用查** |

### 答辩要点

- 无状态——服务器不存登录信息，Token 自带用户身份
- Header 传输——不受 Cookie 跨域限制
- 24 小时过期——降低泄露风险
- HS256 签名——篡改后签名不对，自动识别

## 五、MySQL

### 是什么

开源关系型数据库。所有数据（用户、活动、报名记录……）都存在这里。

### 项目里的 7 张表

| 表名 | 记录什么 | 行数 |
|------|----------|:--:|
| `users` | 用户账号、密码（加密）、角色 | 6 |
| `categories` | 活动分类 | 5 |
| `activities` | 活动信息 | 26 |
| `registrations` | 报名记录 | 若干 |
| `sign_ins` | 签到签退记录 | 若干 |
| `reviews` | 评价内容 + 评分 | 若干 |
| `notices` | 系统公告 | 2 |

### 数据库设计要点

- **字符集 utf8mb4**：支持 emoji 和所有中文，不会乱码
- **引擎 InnoDB**：支持事务和外键
- **命名规范**：表名复数小写（`users`），列名 snake_case（`real_name`）
- **索引设计**：外键列（`category_id`、`organizer_id`）、状态列（`status`）、时间列（`start_time`）建索引，加快查询

### 答辩要点
- 7 张表覆盖完整业务，表关系清晰
- utf8mb4 字符集防中文乱码
- InnoDB 引擎保证事务一致性

## 六、Jackson + jsr310

### 是什么

Jackson 是 Java 的 JSON 处理库——把 Java 对象转成 JSON 字符串（序列化），把 JSON 字符串转成 Java 对象（反序列化）。jsr310 是它的时间模块扩展。

### 项目里做了什么

前端发请求带 JSON 数据，后端返回 JSON 数据。Jackson 负责中间的自动转换：

```
前端：{ "title": "活动标题", "startTime": "2026-06-15 14:00:00" }
        ↓ @RequestBody + Jackson 自动转换
后端：ActivitySaveDTO { title="活动标题", startTime=LocalDateTime(...) }

后端：Result { code=200, data=Activity对象 }
        ↓ Jackson 自动转换
前端：{ "code": 200, "data": { "title": "活动标题", ... } }
```

### 踩过的坑：LocalDateTime 反序列化

Java 的 `LocalDateTime` 默认期望 ISO 格式 `2026-06-15T14:00:00`（带 T），但前端发的格式是 `2026-06-15 14:00:00`（空格）。不加配置的话，后端收到就报错：`DateTimeParseException: Text could not be parsed at index 10`。

**解决方案（项目里三种方式都用上了）**：
1. `pom.xml` 补加 `jackson-datatype-jsr310` 依赖——让 Jackson 认识 LocalDateTime
2. `JacksonConfig.java` 全局配置：`LocalDateTimeSerializer` + `LocalDateTimeDeserializer`，格式统一为 `yyyy-MM-dd HH:mm:ss`
3. `ActivitySaveDTO.java` 字段加 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`——兜底

### 答辩要点
- Jackson 自动完成 Java ↔ JSON 双向转换，不用手写
- 时间格式兼容是前后端协作的常见坑，三层防护保障

## 七、Lombok

### 是什么

Java 的代码简化库。用注解代替手写 getter、setter、构造器等样板代码。

### 为什么用它

**不用 Lombok**：

```java
public class User {
    private Long id;
    private String username;

    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    // 7 个实体 × 平均 8 个字段 × 2 个方法 = 112 个方法，全是重复代码
}
```

**用 Lombok**：
```java
@Data  // 自动生成所有 getter/setter/toString/equals/hashCode
public class User {
    private Long id;
    private String username;
    // 没了，Lombok 编译期自动生成所有方法
}
```

### 项目里用的注解

| 注解 | 作用 | 用于 |
|------|------|------|
| `@Data` | 生成 getter/setter/toString/equals/hashCode | 所有 entity 和 dto |
| `@Slf4j` | 自动创建日志对象 `log` | `GlobalExceptionHandler` |

### 答辩要点
- 减少 80% 样板代码，类更干净
- 编译期生成，不影响运行时性能
- 字段增删时不用手动同步 getter/setter

## 八、Hutool

### 是什么

国产 Java 工具库。提供日期处理、类型转换、Bean 拷贝、加密等常用工具方法。相当于 Java 版的"瑞士军刀"。

### 项目里怎么用

| 方法 | 用到的地方 | 替代了什么 |
|------|-----------|-----------|
| `BeanUtil.copyProperties(dto, activity)` | `ActivityServiceImpl.createActivity()` | 不用逐字段 `activity.setTitle(dto.getTitle())`，一行拷完 |
| `BeanUtil.copyProperties(dto, user)` | `UserServiceImpl.register()` | 同上 |
| `DateUtil` | 各处日期处理 | 不用繁琐的 `SimpleDateFormat` |

**举例**——不用 Hutool vs 用 Hutool：

```java
// 不用 Hutool：DTO 转 Entity，每个字段手工赋值
activity.setTitle(dto.getTitle());
activity.setDescription(dto.getDescription());
activity.setCategoryId(dto.getCategoryId());
activity.setLocation(dto.getLocation());
activity.setStartTime(dto.getStartTime());
activity.setEndTime(dto.getEndTime());
activity.setMaxParticipants(dto.getMaxParticipants());
// 7 行，字段一多容易漏

// 用 Hutool：
BeanUtil.copyProperties(dto, activity);
// 1 行，字段自动匹配
```

### 答辩要点
- 国产开源工具库，文档中文
- BeanUtil 一行代替 N 行属性拷贝
- 减少重复造轮子

## 九、技术协作全景图

```
用户浏览器（Vue 3）
    │  HTTP POST /api/user/login
    │  Body: { "username": "student01", "password": "123456" }
    │  Content-Type: application/json
    ↓
Spring Boot（Tomcat :8080）
    │
    ├─ CORS 过滤器（CorsConfig）→ 允许跨域
    ├─ JWT 过滤器（JwtAuthFilter）→ 本次请求还没 Token，先放行
    ├─ Spring Security（SecurityConfig）→ 这个接口是 permitAll，放行
    │
    ├─ UserController.login()
    │   └─ @Valid @RequestBody → Jackson 把 JSON 转成 LoginDTO
    │       └─ userService.login(dto)
    │           ├─ 查数据库 → MyBatis-Plus UserMapper
    │           ├─ BCrypt 校验密码
    │           └─ JwtUtil.generateToken() → 生成 JWT
    │
    ├─ 返回 Result { code:200, data: { token:"...", role:"student" } }
    │   └─ Jackson 把 Result 转成 JSON 字符串
    │
    ↓
前端收到 Token → localStorage → 跳首页

=== 后续请求（已登录）===

前端 Axios → Header: Authorization: Bearer <token>
    ↓
JwtAuthFilter → 解析 Token → 验证 → userId=4, role=student → SecurityContext
    ↓
SecurityConfig → 检查路径权限 → 有权限 → 放行
    ↓
Controller → SecurityUtil.getCurrentUserId() → 拿到 4
    ↓
Service → 业务处理
    ↓
Mapper → SQL → MySQL
    ↓
返回 Result → Jackson 序列化 → 前端接收
```
