## 关键编码规范

- 后端统一返回格式：`Result<T>` 类，code + message + data
- 通过 `SecurityUtil.getCurrentUserId()` 获取当前登录用户 ID
- MyBatis-Plus 用 `LambdaQueryWrapper` 构建查询条件
- 前端 API 拦截器：`request.js` 中 `baseURL: '/api'`，自动挂 Token
- 登录页路由设为公开（`meta: { noAuth: true }`），其他需要登录守卫
- 时间格式统一用 `yyyy-MM-dd HH:mm:ss`，DTO 中 LocalDateTime 字段需要 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`

## 已知坑点（务必注意）

1. **LocalDateTime 反序列化**：pom.xml 必须加 `jackson-datatype-jsr310` 依赖，否则前端发的时间后端解析失败
2. **测试数据密码**：bcrypt 哈希必须用真正的 hash，不能随便写——可以用 Python `bcrypt.hashpw(b'123456', bcrypt.gensalt())` 生成
3. **数据库字符集**：用 `utf8mb4`，否则中文乱码
4. **Vite 代理**：`vite.config.js` 配置 proxy，`/api` → `localhost:8080`
5. **预约冲突检测**：同一座位同一时段不能重复预约，需要在后端做校验（查已有预约的时间重叠）
6. **CORS 跨域**：后端需要 CorsFilter 配置 + Spring Security 放行
