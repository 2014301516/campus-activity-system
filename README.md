# 校园活动管理系统

Campus Activity Management System

一个面向课程设计演示的校园活动管理系统，包含学生报名签到、组织者发布活动、管理员审核与后台管理等完整流程。当前版本重点优化了首页展示、活动状态自动流转、角色权限边界，以及后台管理的演示稳定性。

## 项目亮点

- 首页采用推荐活动、即将开始、新上架活动、活动广场的组合布局，适合答辩演示。
- 活动支持封面图 URL，首页和详情页均可展示活动图片。
- 活动状态支持按时间自动流转：`approved -> ongoing -> ended`。
- 学生端支持报名、取消报名、签到、签退、提交评价。
- 组织者端支持创建活动、修改活动、查看自己活动。
- 管理员端支持活动审核、用户管理、分类管理、公告管理、数据概览。
- 后台已补充演示保护：全量活动分页、禁止禁用自己、禁止删除仍被活动使用的分类。

## 快速启动

### 1. 初始化数据库

打开 MySQL 后执行：

```sql
source E:/keshe/campus-activity-system/docs/schema.sql;
source E:/keshe/campus-activity-system/docs/data.sql;
```

或者使用命令行：

```bash
mysql -u root -p123 < docs/schema.sql
mysql -u root -p123 < docs/data.sql
```

默认数据库配置：

- 地址：`localhost:3306`
- 用户名：`root`
- 密码：`123`
- 数据库名：`campus_activity`

### 2. 启动后端

1. 用 IntelliJ IDEA 打开 `backend/pom.xml`
2. 等待 Maven 依赖下载完成
3. 运行 `CampusActivityApplication`

默认后端地址：

- `http://localhost:8080`

### 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

默认前端地址：

- `http://localhost:5173`

说明：

- 前端已配置 Vite 代理，`/api` 会自动转发到 `http://localhost:8080`

## 角色说明

### 学生

- 浏览首页、活动广场、活动详情
- 报名和取消报名
- 在允许时间内签到/签退
- 对参加过的活动提交评价
- 查看自己的报名记录

### 组织者

- 创建活动
- 编辑和删除自己创建的活动
- 为活动填写封面图、时间、地点、人数等信息
- 查看自己名下的活动并提交审核

### 管理员

- 进入后台管理页面
- 审核活动
- 查看数据概览
- 管理用户状态
- 查看全部活动
- 管理活动分类
- 发布和删除公告

## 活动状态流转

当前活动状态包括：

- `draft` 草稿
- `pending` 待审核
- `approved` 已通过 / 可报名
- `rejected` 已驳回
- `ongoing` 进行中
- `ended` 已结束
- `cancelled` 已取消

流转规则：

```text
组织者创建活动 -> pending
管理员审核通过 -> approved
管理员审核驳回 -> rejected
rejected 状态下再次编辑 -> pending
活动开始时间到达 -> approved 自动变为 ongoing
活动结束时间到达 -> approved / ongoing 自动变为 ended
```

## 当前首页结构

首页主要由以下模块构成：

- 首屏推荐区
- 最新公告
- 即将开始
- 新上架活动
- 活动广场（分类、关键词、排序、分页）

其中：

- `浏览全部活动` 会重置筛选并滚动到活动广场
- `查看热门活动` 会切换热门排序并滚动到活动广场
- 首屏右侧推荐卡也支持展示封面图

## 后台管理说明

管理员后台当前包含：

- 数据概览
- 用户管理
- 活动审核
- 全部活动
- 分类管理
- 公告管理

已补充的后台保护：

- 全部活动为真正的全量视图，支持分页，不再默认只看首页可见活动
- 禁用用户时不能禁用当前登录管理员
- 禁用用户时不能禁用最后一个可用管理员
- 删除分类前会检查是否仍有活动使用该分类

## API 返回格式

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

前端通过 Axios 统一处理业务错误和未登录状态。

## 适合演示的功能路径

推荐按以下顺序演示：

1. 首页浏览推荐活动和活动广场
2. 学生登录后报名、签到、评价
3. 组织者登录后创建活动并提交审核
4. 管理员登录后审核活动、查看后台统计、管理公告

这样能够完整覆盖三种角色和平台主流程。
